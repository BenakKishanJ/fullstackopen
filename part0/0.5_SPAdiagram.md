# 0.5: Single page app diagram

```mermaid
sequenceDiagram
    participant browser
    participant server

    browser->>server: GET /spa
    activate server
    server-->>browser: HTML document (SPA page)
    deactivate server

    browser->>server: GET /main.css
    server-->>browser: CSS file

    browser->>server: GET /spa.js
    server-->>browser: JavaScript file

    Note right of browser: Browser executes spa.js

    browser->>server: GET /data.json
    activate server
    server-->>browser: Notes data (JSON)
    deactivate server

    Note right of browser: JS renders notes dynamically using DOM
````

## Explanation

When the user opens the SPA version of the notes app, the browser first requests the HTML page from the server. After receiving the HTML, it fetches the CSS and JavaScript files. Once the JavaScript (spa.js) is loaded, it runs in the browser and makes an additional request to fetch the notes data in JSON format from `/data.json`.

Unlike traditional web pages, the browser does not receive fully rendered HTML for the notes. Instead, the JavaScript code processes the JSON data and dynamically updates the DOM to display the notes. This is the core idea of a Single Page Application: the page is loaded once, and further updates happen dynamically without full page reloads.
