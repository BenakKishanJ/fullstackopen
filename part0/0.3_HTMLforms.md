# HTML Forms Notes — Your First Form (MDN Review Notes)

## 1. Purpose of Web Forms

Web forms are used to **collect user input and send it to a server** for processing or use it on the client side. ([MDN Web Docs][1])

* Primary interaction point between user and website
* Collect structured data (name, email, message, etc.)
* Data is typically sent as **name/value pairs**

### Key Idea

A form consists of:

* **Form controls (widgets)** → input fields, buttons, etc.
* **Structure elements** → labels, containers

---

## 2. Designing a Form (Conceptual Step)

Before writing code:

* Decide what data you actually need
* Keep the form simple to avoid user frustration ([MDN Web Docs][1])
* Plan layout (e.g., contact form with name, email, message)

### Example Structure

* Name (text input)
* Email (email input)
* Message (textarea)
* Submit button

---

## 3. The `<form>` Element

All forms start with the `<form>` element.

### Basic Syntax

```html
<form action="/my-handling-form-page" method="post">
  ...
</form>
```

### Important Attributes

* `action` → URL where data is sent
* `method` → HTTP method (`get` or `post`) ([MDN Web Docs][1])

### Key Concept

* Acts as a **container for all form controls**
* Defines **how and where data is submitted**

---

## 4. Form Controls: `<label>`, `<input>`, `<textarea>`

### Example Implementation

```html
<form action="/my-handling-form-page" method="post">
  <p>
    <label for="name">Name:</label>
    <input type="text" id="name" name="user_name" />
  </p>

  <p>
    <label for="mail">Email:</label>
    <input type="email" id="mail" name="user_email" />
  </p>

  <p>
    <label for="msg">Message:</label>
    <textarea id="msg" name="user_message"></textarea>
  </p>
</form>
```

### Key Elements

#### `<label>`

* Describes the purpose of an input
* Connected using:

```html
<label for="name">Name:</label>
<input id="name" />
```

* Improves accessibility and usability ([MDN Web Docs][1])

#### `<input>`

* Used for single-line input fields
* Controlled by `type` attribute

Examples:

```html
<input type="text" />
<input type="email" />
```

* `type="email"` enforces email format validation ([MDN Web Docs][1])

#### `<textarea>`

* Used for multi-line input

```html
<textarea></textarea>
```

---

## 5. Input Types and Behavior

### Common Types Used

* `text` → general text input
* `email` → validates email format

### Key Insight

The `type` attribute determines:

* Input behavior
* Validation rules
* UI (e.g., mobile keyboard layout) ([MDN Web Docs][1])

---

## 6. Default Values in Inputs

### For `<input>`

```html
<input type="text" value="Default text" />
```

### For `<textarea>`

```html
<textarea>
Default text
</textarea>
```

### Important Difference

* `<input>` → uses `value` attribute
* `<textarea>` → content inside tags ([MDN Web Docs][1])

---

## 7. The `<button>` Element

### Adding Submit Button

```html
<p class="button">
  <button type="submit">Send your message</button>
</p>
```

### Button Types

* `submit` → sends form data (default)
* `reset` → resets form fields (generally avoided)
* `button` → no default behavior ([MDN Web Docs][1])

### Key Concept

* Triggers form submission to the `action` URL

---

## 8. Naming Form Data (`name` Attribute)

Each input must have a `name` attribute:

```html
<input type="text" name="user_name" />
```

### Why It Matters

* Data is sent as:

```
user_name = value
user_email = value
```

* Server reads data using these names ([MDN Web Docs][1])

---

## 9. Sending Data to Server

### Example Form

```html
<form action="/my-handling-form-page" method="post">
  ...
</form>
```

### How It Works

* Browser collects all inputs with `name`
* Sends data as key-value pairs
* Server processes the request

### Example Data Sent

```
user_name=John
user_email=john@email.com
user_message=Hello
```

## 11. Basic Form Styling (CSS Overview)

```css
form {
  margin: 0 auto;
  width: 400px;
  padding: 1em;
  border: 1px solid #ccc;
  border-radius: 1em;
}

label {
  display: inline-block;
  width: 90px;
  text-align: right;
}

input,
textarea {
  width: 300px;
  border: 1px solid #999;
}

textarea {
  height: 5em;
}
```

### Key Styling Concepts

* Center form using `margin: 0 auto`
* Align labels using fixed width
* Maintain consistent input sizes
* Improve usability with spacing

## 12. Key Concepts Summary

### Core Ideas

* Forms collect and send user data
* `<form>` defines submission behavior
* `<input>`, `<textarea>` collect data
* `<label>` improves accessibility
* `name` attribute defines data keys
* `<button>` triggers submission

## 13. Conceptual Structure

```
Form
│
├── Structure
│   └── form (action, method)
│
├── Inputs
│   ├── input (text, email)
│   ├── textarea
│   └── button
│
├── Accessibility
│   └── label (for + id)
│
└── Data Flow
    ├── name attributes
    └── key-value pairs sent to server
```

## 14. Final Summary

* Forms are essential for user interaction
* Built using a combination of:

  * Structure (`form`)
  * Controls (`input`, `textarea`)
  * Labels (`label`)
  * Actions (`button`)
* Data submission depends on:

  * `action` (where)
  * `method` (how)
  * `name` (what data)
