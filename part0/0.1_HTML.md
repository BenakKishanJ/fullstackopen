# HTML Notes — Creating the Content (MDN Review Notes)

## 1. Purpose of HTML

HTML (HyperText Markup Language) is used to **structure web page content**. It wraps text using elements so that browsers can interpret and display it meaningfully. ([MDN Web Docs][1])

* Converts raw text into structured content (paragraphs, lists, etc.)
* Defines meaning, not presentation
* Enables additional features like links, images, and media

### Example: Unstructured vs Structured Content

```html
Instructions for life:
Eat
Sleep
Repeat
```

```html
<p>Instructions for life:</p>

<ul>
  <li>Eat</li>
  <li>Sleep</li>
  <li>Repeat</li>
</ul>
```

---

## 2. HTML Elements and Syntax

HTML consists of **elements**, which are the building blocks of a webpage.

### General Structure

```html
<tagname attribute="value">Content</tagname>
```

### Key Components

* **Opening tag**: `<tagname>`
* **Content**: Text or nested elements
* **Closing tag**: `</tagname>`
* **Attributes**: Provide additional information

### Empty (Void) Elements

Some elements do not have closing tags:

```html
<img src="image.png" alt="description" />
```

* No content inside
* Closing slash is optional but commonly used ([MDN Web Docs][1])

---

## 3. Basic HTML Document Structure

A complete HTML page is formed by combining multiple elements.

### Boilerplate Structure

```html
<!doctype html>
<html lang="en-US">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width" />
    <title>My test page</title>
  </head>
  <body>
    <img src="" alt="My test image" />
  </body>
</html>
```

### Key Sections

* `<!doctype html>`: Declares HTML version
* `<html>`: Root element
* `<head>`: Metadata (not visible)
* `<body>`: Visible content

---

## 4. Adding Images

Images are embedded using the `<img>` element.

### Syntax

```html
<img src="images/firefox-icon.png" alt="Firefox logo" />
```

### Important Points

* `src`: Path to the image file
* `alt`: Text description (important for accessibility and fallback)
* Image path is relative to the HTML file location ([MDN Web Docs][1])

---

## 5. Marking Up Text

HTML provides elements to structure textual content.

### Headings

```html
<h1>Main heading</h1>
<h2>Subheading</h2>
<h3>Smaller heading</h3>
```

* Range: `<h1>` to `<h6>`
* Defines hierarchical structure

### Paragraphs

```html
<p>This is a paragraph.</p>
```

### Lists

#### Unordered List

```html
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>
```

#### Ordered List

```html
<ol>
  <li>First</li>
  <li>Second</li>
</ol>
```

* `<ul>`: Bullet points
* `<ol>`: Numbered sequence
* `<li>`: List item

---

## 6. Creating Links

Links connect one resource to another using the `<a>` (anchor) element.

### Syntax

```html
<a href="https://www.example.com">Visit website</a>
```

### Key Points

* `href`: Destination URL
* Must include full URL (including protocol like `https://`)
* Can link to external or internal pages ([MDN Web Docs][1])

---

## 7. Combining Elements into a Web Page

HTML elements are combined to form a complete webpage.

### Example Page Content

```html
<h1>Mozilla is cool</h1>

<img src="images/firefox-icon.png" alt="Firefox logo" />

<p>At Mozilla, we’re a global community of</p>

<ul>
  <li>technologists</li>
  <li>thinkers</li>
  <li>builders</li>
</ul>

<p>working together to keep the Internet alive and accessible.</p>

<a href="https://www.mozilla.org/en-US/about/manifesto/">
  Mozilla Manifesto
</a>
```

---

## 8. Key Takeaways

* HTML structures content using elements
* Each element has a specific semantic meaning
* Core elements covered:

  * Text: headings, paragraphs
  * Lists: ordered and unordered
  * Media: images
  * Navigation: links
* Proper structure improves readability and accessibility

---

## 9. Conceptual Summary

HTML works as a **layered structure system**:

```
Document
 ├── Head (metadata)
 └── Body (content)
      ├── Text (headings, paragraphs)
      ├── Lists
      ├── Images
      └── Links
```
