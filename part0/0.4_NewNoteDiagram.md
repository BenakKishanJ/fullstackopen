# 0.4: New note diagram

```mermaid
sequenceDiagram
    participant browser
    participant server

    Note right of browser: User enters note and clicks "Save"

    browser->>server: POST /new_note (form data)
    activate server

    Note right of server: Reads req.body.note<br/>Creates new note object<br/>Adds to notes array

    server-->>browser: 302 Redirect (Location: /notes)
    deactivate server

    Note right of browser: Browser follows redirect and reloads page

    browser->>server: GET /notes
    activate server
    server-->>browser: HTML document
    deactivate server

    browser->>server: GET /main.css
    server-->>browser: CSS file

    browser->>server: GET /main.js
    server-->>browser: JavaScript file

    Note right of browser: JS runs and fetches notes data

    browser->>server: GET /data.json
    server-->>browser: Updated notes JSON

    Note right of browser: Browser renders updated notes list
````

## Explanation

When the user submits a new note, the browser sends a POST request to `/new_note` with the form data. The server processes this request by extracting the note content from the request body, creating a new note object, and adding it to the notes array.

Instead of returning the updated page directly, the server responds with a **302 redirect**, instructing the browser to make a new GET request to `/notes`. This causes the entire page to reload.

During the reload, the browser fetches the HTML document, CSS, and JavaScript files again. Once the JavaScript executes, it sends an additional request to `/data.json` to retrieve the updated list of notes, which now includes the newly added note. Finally, the browser renders the updated notes on the page.
